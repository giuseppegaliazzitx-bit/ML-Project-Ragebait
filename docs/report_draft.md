## 1. Introduction
As digital platforms optimize for user engagement, they inadvertently incentivize the creation of "ragebait"—manipulative content strategically engineered to provoke anger, outrage, or frustration to artificially inflate interaction metrics. Detecting this content is a complex natural language processing (NLP) challenge, as ragebait often avoids explicit profanity or traditional toxic markers in favor of subtle linguistic cues and implicit emotional triggers. The goal of this project is to develop a machine learning pipeline capable of differentiating ragebait and trolling from organic, benign discourse.

Our initial attempt at solving this problem framed it as a massive, multi-source binary classification task using over 500,000 weak-labeled rows generated by vLLM. While our models in this first iteration achieved high accuracy (e.g., 0.8767 for a tuned BERT classifier), we realized a critical flaw: measuring agreement with an LLM is not the same as establishing human-grounded truth. This key finding perfectly motivated our pivot towards a more rigorous approach utilizing human-annotated gold standard data.

We approach this problem by evaluating three distinct tiers of machine learning models: Tier 1 (Classical TF-IDF models), Tier 2 (Feed-Forward Neural Networks), and Tier 3 (Fine-tuned BERT Transformers). By training these models on a highly curated, human-annotated dataset, we aim to measure the performance gains achieved by moving from sparse lexical representations to deep contextual embeddings. 

Our experiments reveal that while classical models perform admirably on binary classification (Ragebait vs. Normal), they struggle significantly with nuanced multi-class categorization. The fine-tuned BERT architecture achieved state-of-the-art performance across both tasks (0.925 F1 for binary, 0.640 Macro F1 for multi-class), proving that bidirectional attention mechanisms are essential for capturing the bad-faith framing inherent to sophisticated trolling.

## 2. Data
The primary data source for this project is "Trawling for Trolling: A Dataset," a human-annotated corpus containing 12,490 samples. The dataset differentiates offensive content based on the severity of the offense and encompasses data from Twitter, Reddit, and Wikipedia Talk Pages. 

This dataset serves as a significant upgrade from our Iteration 1 dataset. The initial mixed-source corpus had an extreme class imbalance, containing only 6.12% ragebait, which required heavy synthetic balancing just to train a stable model. In contrast, the `trolldata.csv` dataset provided a much healthier natural distribution and nuanced 5-class human annotations, making it the superior choice for our final architecture.

To prevent data leakage, we established a canonical, stratified 80/10/10 split across all experiments, resulting in 9,992 training rows, 1,249 validation rows, and 1,249 test rows. 

For Experiment 1 (Binary Classification), the dataset was mapped into two classes: Normal (40.4%) and Ragebait (59.6%), the latter combining all offensive sub-categories. For Experiment 2 (Multi-Class Classification), we utilized the original 5-class schema. The class distribution reflects real-world imbalances: Normal (40.4%), Trolling (36.3%), Profanity (12.6%), Derogatory (6.9%), and Hate Speech (3.6%).

## 3. Methodology
To evaluate the progression of model capacity, we implemented a Three-Tier architecture:
* **Tier 1 (Classical Baselines):** Logistic Regression and Linear SVC utilizing TF-IDF vectorization. These provide a fast, interpretable lexical baseline.
* **Tier 2 (Intermediate Neural Network):** A PyTorch-based Feed-Forward Neural Network (FFNN). This model maps tokens to embedding vectors and averages them into a pooled representation before applying non-linear activation functions. 
* **Tier 3 (Transformer):** A fine-tuned `bert-base-uncased` sequence classifier. Unlike the FFNN's bag-of-embeddings approach, BERT utilizes bidirectional self-attention to contextualize every token based on its surrounding sequence, allowing it to capture subtle semantic manipulation.

**Handling Class Imbalance:**
Because the multi-class dataset is highly imbalanced (e.g., Hate Speech constitutes only 3.6% of the data), standard loss functions would cause the models to collapse into predicting only the majority classes. We mitigated this by computing inverse-frequency class weights: Normal (0.49), Profanity (1.57), Trolling (0.55), Derogatory (2.89), and Hate Speech (5.47). These weights were directly injected into the PyTorch `CrossEntropyLoss` criterion, penalizing the model heavily for missing minority class predictions.

## 4. Implementations
The project was built using Python, leveraging `scikit-learn` for data splitting, TF-IDF vectorization, and Tier 1 model execution. The Tier 2 and Tier 3 models were implemented natively in `PyTorch`, utilizing the `Transformers` library from HuggingFace for BERT tokenization and sequence classification.

Several modern PyTorch engineering standards were implemented to ensure reproducibility and efficiency:
1. **Mixed Precision Training:** We utilized `torch.amp.GradScaler` to accelerate GPU training and reduce memory footprint without sacrificing gradient accuracy.
2. **Deterministic Execution:** We locked down random seeds globally and forced PyTorch to use deterministic algorithms (disabling non-deterministic CUDA SDP kernels) to ensure all runs produce mathematically identical artifacts.
3. **Metric-Driven Checkpointing:** Instead of saving checkpoints based on raw accuracy, the multi-class training loop dynamically saved the best model state evaluated strictly on Validation Macro-F1.

### Error Analysis
To understand BERT's limitations, we built an automated diagnostic pipeline to extract the highest-confidence false predictions using Softmax probability logic. We discovered two primary modes of failure:

1. **Trolling vs. Derogatory Boundary Blur:** The most frequent high-confidence errors occurred between these two classes. For example, the text *"Douchebag is a washed up has been who is mostly a crackhead bitch"* was predicted by BERT as **Derogatory** with 94.6% confidence, but the human gold-label was **Trolling**. Both categories rely on insults, making the boundary highly subjective even for human annotators. The model struggles to separate a mild-mannered insult intended for amusement (Trolling) from a direct, highly offensive attack (Derogatory).
2. **Profanity vs. Hate Speech (Implicit Harm):** In another instance, BERT predicted *"get fucking cancer"* as **Profanity** with 96.3% confidence, completely missing the true **Hate Speech** label. The model successfully recognized the explicit profane token but failed to semantically map the phrase to an extreme threat/wish of harm. This indicates that while BERT is context-aware, highly colloquial or indirect threats can still bypass its safety parameters.

### Speed Analysis
A critical finding during this project was the computational bottleneck of transformer architectures on standard hardware. During Experiment 1, the BERT model was inadvertently trained utilizing the CPU. The training loop took **4,893 seconds** (~1.3 hours), with a prediction inference time of **64.8 seconds** for the 1,249-row validation set.

For Experiment 2, device logic was corrected to utilize a CUDA-enabled GPU. Despite tackling a vastly more complex 5-class prediction problem, GPU acceleration reduced BERT's training time to just **376.7 seconds** (~6 minutes) and the validation prediction time to **9.86 seconds**. 
* **Inference Cost:** On GPU, the BERT multi-class inference cost is exceptionally low, averaging just **7.89 milliseconds per example** (9.86s / 1249 samples), proving that deploying this model for real-time social media filtering is highly feasible on modern hardware. 

### Retrospective: Weak Labels vs. Gold Labels
As noted in our introduction, our first iteration tackled this problem using over 500,000 weak-labeled samples generated by an LLM. Our Iteration 1 tuned-BERT model achieved a respectable 0.8767 accuracy on these weak labels. However, when transitioning to the Iteration 2 Binary BERT trained on human-annotated gold labels, the model achieved a 0.9111 accuracy. This comparison highlights a fundamental lesson in machine learning data engineering: training models on clean, human-annotated data yields a much more confident and accurate classifier than relying on noisy, LLM-generated weak labels.

## 6. Conclusion
This project successfully demonstrates that contextualized transformer models like BERT are highly effective at detecting adversarial social media content, significantly outperforming classical algorithms. However, deep error analysis reveals that subjective boundaries between categories like "Trolling" and "Derogatory" remain challenging, even for bidirectional models.
